Frequently Asked Questions (FAQ)

What is an Avail Announcement file?

An Avail Announcement file is an MS Excel report downloaded from WB’s SFDC (Sales Force Dot Com) application. It contains the title information of all marketing and promotions of WBs that are launched till date.

 

What is an Apple QC report?

An Apple QC report is a file received from Apple iTunes. It contains cumulative data of all the titles specific to WB, including quality status and date of import.

 

What is an Apple Price report?

An Apple Price report is a file received from Apple iTunes. It contains cumulative data of the titles, along with territories and formats that are live on Apple iTunes.

 

What are Pipeline orders?

Pipeline Orders is a file received from the TOPs order management system. It contains the status of orders in TOPs along with the MR No. The pipeline order connects the orders between HAL and Deluxe Avails. This data is cumulative.

 

What are the actionable columns in an Avail Announcement file?

Client Start date, Client End date, and Client Avail Status are the actionable columns. Any changes made in these columns results in either advancement or postponement of a title.

 

When is a title considered to be cancelled?

For a given title, if the statuses of all the formats (VOD SD, VOD HD, EST SD, EST HD) are blank or cancelled, then the title is considered to be cancelled.

 

What is the frequency of an Avail announcement file?

The Avail announcement file is generated on a weekly basis.

 

In what scenarios is a new order created?

New orders are created in the following scenarios:

·       A title is being launched for the first time.

·       A title is being launched in a new country.

·       A title is being launched in a language for the first time.

·       The Assets required for launching a title are not available.

 

What is the output of reports?

The output of reports is directly exported to an MS Excel file.

 

Under what circumstances does a job fail?

A job fails when:

·       The file (Announcement file, Apple Price Report, Apple QC Report, or Pipeline Orders) is not available at the desired location.

·       A file is of an incorrect format.

·       You are not able to log on to the FTP site.

 

What happens if an import job is not run as per the recommended sequence?

The recommended sequence to run an import job is as follows:

Pipeline Orders → Apple Price Report → Apple QC Report → Announcement File

It is mandatory to run the import Announcement job once the remaining import jobs have been run. If you do not follow the recommended order, then at the time of processing the Announcement file, all the import jobs that are run after the Announcement file, will not be considered for processing.

For example if the user follows the following sequence:

Apple Price Report → Announcement File → Apple QC Report → Pipeline Orders

Then at the time of processing the Announcement file, all the date imported from the Apple QC Report and Pipeline Orders is not considered since these files were imported after the Announcement file.

 

What is the impact of running the import Announcement job first? Would that corrupt the data?

The files (Apple Price Report, Apple QC Report, and Pipeline Orders) available on the FTP server are updated every week. Processing of the Announcement file depends on the updates occurred on these files in the previous week. If there is any mismatch in the data of the last week, in comparison to the current week, then there is a greater chance of the data getting corrupted. (Announcement file not getting processed with the latest data).

 

During a job failure, I am not receiving an email alert? How can I rectify this?

·       Under the notification section, ensure that the Notification check box is checked, and an email ID is provided against the appropriate event names.

·       Check with the IT team for any issues with the SMTP server.

 

How can I trace the cause of a job failure?

When a job fails, a log is generated, please refer to it for details.
